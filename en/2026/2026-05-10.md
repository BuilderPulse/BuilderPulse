### What are the most important open-source AI developments this week?

**🔍 Signal**: Open AI development centered on agent work control: [mattpocock/skills](https://github.com/mattpocock/skills), [browserbase/skills](https://github.com/browserbase/skills), [openai/symphony](https://github.com/openai/symphony), [9router](https://github.com/decolua/9router), [InsForge](https://github.com/InsForge/InsForge), and DEV posts about Model Context Protocol (MCP) schema drift and unauthenticated servers.

**In plain English:** The open-source frontier is no longer just models; it is the plumbing that decides what AI tools may do.

Model Context Protocol, or MCP, is a way for AI apps to call tools and read external resources. That makes schemas, authentication, and permissions load-bearing. DEV’s [Schema descriptions are load-bearing](https://dev.to/incultnitollc/schema-descriptions-are-load-bearing-why-missing-parameter-descriptions-break-mcp-clients-4l42) argues that missing parameter descriptions break clients. [MCP Sentinel v1.0](https://dev.to/wannavf/mcp-sentinel-v10-is-out-a-lockfile-for-mcp-tool-schemas-3hb5) packages schema drift as a lockfile problem. A separate DEV post claims 38% of MCP servers have no authentication.

The GitHub list says the same thing at a higher level. [openai/symphony](https://github.com/openai/symphony) isolates implementation runs. [9router](https://github.com/decolua/9router) routes coding tools across model providers. [InsForge](https://github.com/InsForge/InsForge) gives coding agents backend primitives. These projects are infrastructure, but the paid layer is governance: who approved the tool, which schema changed, what data could leave, and what bill was created.

That makes “open-source AI” less like the old model-release cycle and more like DevOps. A useful product can be tiny if it sits at a control point: compare two schemas, flag missing auth, show which tool can read which resource, or explain why a routing rule sent a task to an expensive model. The repo is the demo; the report is what teams buy.

**Takeaway**: Build control layers around open AI plumbing; schema locks, auth scans, and routing reports are more defensible than another prompt pack.

**Counter-view**: Standards and platform vendors may absorb the best ideas if small tools cannot integrate quickly.
